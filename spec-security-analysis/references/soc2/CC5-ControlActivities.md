# CC5 - Control Activities

**Category**: Common Criteria  
**Focus**: Design and deployment of controls

## When to Apply

Apply CC5 when your component:
- Implements security controls
- Uses technology to enforce controls
- Has documented policies/procedures

## Controls

| Control | Name | Description |
|---------|------|-------------|
| CC5.1 | Control Selection | Entity selects and develops control activities that contribute to risk mitigation |
| CC5.2 | Technology Controls | Entity selects and develops general control activities over technology |
| CC5.3 | Policy Deployment | Entity deploys control activities through policies that establish expectations |

## Mapping Guidance

For software components:
- **Access control implementation** → CC5.1, CC5.2
- **Input validation** → CC5.2
- **Configuration management** → CC5.2
- **Documentation** → CC5.3
- **Code review requirements** → CC5.3
