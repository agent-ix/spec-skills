# A.6 - People Controls

**Theme**: People  
**Controls**: 8  
**Focus**: Personnel security throughout employment lifecycle

## When to Apply

A.6 controls are primarily HR/organizational. Software components typically inherit these controls.

## Controls

| Control | Name | Description | Software Relevance |
|---------|------|-------------|-------------------|
| A.6.1 | Screening | Background verification | LOW - HR process |
| A.6.2 | Terms and Conditions of Employment | Security in contracts | LOW - HR process |
| A.6.3 | Information Security Awareness, Education and Training | Security training | LOW - Training |
| A.6.4 | Disciplinary Process | Security violation handling | LOW - HR process |
| A.6.5 | Responsibilities After Termination or Change of Employment | Post-employment obligations | LOW - HR process |
| A.6.6 | Confidentiality or Non-Disclosure Agreements | NDAs | LOW - Legal |
| A.6.7 | Remote Working | Secure remote work | MEDIUM - Remote auth |
| A.6.8 | Information Security Event Reporting | Event reporting | **HIGH** - Error/incident reporting |

## Mapping Guidance

For software components:
- **A.6.7** (Remote Working) → Auth requirements for remote access
- **A.6.8** (Event Reporting) → FR for security event logging, error reporting

Most A.6 controls are out of scope for software, assigned to HR/Management.
